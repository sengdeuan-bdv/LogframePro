User Guideຄູ່ມືການນຳໃຊ້

A step-by-step guide with tips and examples for every tab. Switch language with the EN / LA / EN-LA buttons at the top right. ຄູ່ມືແບບເທື່ອລະຂັ້ນຕອນ ພ້ອມຄຳແນະນຳ ແລະ ຕົວຢ່າງ ສຳລັບແຕ່ລະແທັບ. ສະຫຼັບພາສາໄດ້ທີ່ປຸ່ມ EN / LA / EN-LA ມຸມຂວາເທິງ.

Recommended workflowລຳດັບການເຮັດວຽກທີ່ແນະນຳ

  1. Project → enter title, code, description.ໂຄງການ → ປ້ອນຊື່, ລະຫັດ, ລາຍລະອຽດ.
  2. Settings → choose number of levels & rename headers.ຕັ້ງຄ່າ → ເລືອກຈຳນວນລະດັບ ແລະ ປ່ຽນຊື່ຫົວຂໍ້.
  3. Logframe → build goals, purposes, outputs, activities & indicators. Everything else flows from here.Logframe → ສ້າງເປົ້າໝາຍ, ຈຸດປະສົງ, ຜົນໄດ້ຮັບ, ກິດຈະກຳ ແລະ ຕົວຊີ້ວັດ. ທຸກແທັບອື່ນດຶງຂໍ້ມູນຈາກບ່ອນນີ້.
  4. Planning / Budget / Tracking / PIRS / Risk → fill in details.ແຜນວຽກ / ງົບປະມານ / ຕິດຕາມ / PIRS / ຄວາມສ່ຽງ → ຕື່ມລາຍລະອຽດ.
  5. Dashboard → review. Save / Export → keep your work.ແຜງຄວບຄຸມ → ກວດເບິ່ງ. ບັນທຶກ / ສົ່ງອອກ → ເກັບວຽກ.

1. Project1. ໂຄງການ (Project)

Purpose:ຈຸດປະສົງ: Record the project's identity — title, code and description (bilingual).ບັນທຶກຂໍ້ມູນຫຼັກຂອງໂຄງການ — ຊື່, ລະຫັດ ແລະ ລາຍລະອຽດ (ສອງພາສາ).

Tip:ຄຳແນະນຳ: The Project Code becomes the default filename when you Download.ລະຫັດໂຄງການຈະກາຍເປັນຊື່ໄຟລ໌ເລີ່ມຕົ້ນເມື່ອກົດ Download.

Example:ຕົວຢ່າງ: Title: "Rural Livelihoods Project" · Code: RLP-2026ຊື່: "ໂຄງການສົ່ງເສີມຊີວິດການເປັນຢູ່ຊົນນະບົດ" · ລະຫັດ: RLP-2026

2. Settings2. ຕັ້ງຄ່າ (Settings)

Purpose:ຈຸດປະສົງ: Set the logframe structure (2–4 levels) and rename the level headers to your terminology.ກຳນົດໂຄງສ້າງ Logframe (2–4 ລະດັບ) ແລະ ປ່ຽນຊື່ຫົວຂໍ້ລະດັບໃຫ້ກົງກັບຄຳສັບຂອງທ່ານ.

Tip:ຄຳແນະນຳ: Do this BEFORE building the Logframe. E.g. rename "Purposes" → "Outcomes".ຄວນເຮັດກ່ອນສ້າງ Logframe. ຕົວຢ່າງ: ປ່ຽນ "ຈຸດປະສົງ" → "ຜົນໄດ້ຮັບຫຼັກ".

Example:ຕົວຢ່າງ: 3 levels = Goals → Outcomes → Outputs (no activities).3 ລະດັບ = ເປົ້າໝາຍ → ຜົນໄດ້ຮັບ → ຜົນຜະລິດ (ບໍ່ມີກິດຈະກຳ).

3. Logframe3. Logframe (ຕາຕະລາງເຫດຜົນ)

Purpose:ຈຸດປະສົງ: The heart of the tool. Build the results chain and its indicators, verification sources and assumptions.ຫົວໃຈຂອງເຄື່ອງມື. ສ້າງລະບົບຕ່ອງໂສ້ຜົນ ພ້ອມຕົວຊີ້ວັດ, ແຫຼ່ງຢັ້ງຢືນ ແລະ ຂໍ້ສົມມຸດຕິຖານ.

How to use:ວິທີໃຊ້:

  1. Click "ໝວດໃໝ່ (Row)" on the Goals header to add a goal.ກົດ "ໝວດໃໝ່ (Row)" ທີ່ຫົວ Goals ເພື່ອເພີ່ມເປົ້າໝາຍ.
  2. Hover a row → green adds a sibling, teal adds a child level.ເລື່ອນເມົ້າໃສ່ແຖວ → ປຸ່ມຂຽວ ເພີ່ມແຖວດຽວກັນ, ປຸ່ມ teal ເພີ່ມລະດັບຍ່ອຍ.
  3. Use "ຂໍ້ໃໝ່ (Item)" to add indicators in the Indicators column.ໃຊ້ "ຂໍ້ໃໝ່ (Item)" ເພື່ອເພີ່ມຕົວຊີ້ວັດໃນຖັນ Indicators.
  4. The yellow filter shows only one goal's branch.ປຸ່ມ ສີເຫຼືອງ ສະແດງສະເພາະສາຍຂອງເປົ້າໝາຍດຽວ.

Tip:ຄຳແນະນຳ: Keep the numbering (1, 1.1, 1.1.1) — Planning, Budget, Tracking & PIRS rely on it.ຮັກສາລະບົບເລກ (1, 1.1, 1.1.1) — ແທັບ ແຜນວຽກ, ງົບປະມານ, ຕິດຕາມ ແລະ PIRS ໃຊ້ມັນ.

Example:ຕົວຢ່າງ: Goal 1: Improve household income → Indicator 1.1: % of households above poverty line.ເປົ້າໝາຍ 1: ຍົກສູງລາຍຮັບຄົວເຮືອນ → ຕົວຊີ້ວັດ 1.1: % ຄົວເຮືອນທີ່ເໜືອເສັ້ນຄວາມທຸກຍາກ.

4. Planning4. ແຜນວຽກ (Planning)

Purpose:ຈຸດປະສົງ: A Gantt chart of activities over time (monthly or weekly).ຕາຕະລາງ Gantt ຂອງກິດຈະກຳຕາມເວລາ (ລາຍເດືອນ ຫຼື ລາຍອາທິດ).

How to use:ວິທີໃຊ້:

  1. Set View (Monthly/Weekly) and the Timeline From / To range.ຕັ້ງ ຮູບແບບ (ລາຍເດືອນ/ລາຍອາທິດ) ແລະ ໄລຍະ ຈາກ / ຫາ.
  2. For each activity, set Start and End dates — the bar draws itself.ໃສ່ວັນທີ ເລີ່ມ ແລະ ສິ້ນສຸດ ຂອງແຕ່ລະກິດຈະກຳ — ແຖບຈະແຕ້ມເອງ.
  3. Status colours the bar: grey=not started, blue=in progress, green=done, red=delayed.ສະຖານະ ປ່ຽນສີແຖບ: ເທົາ=ຍັງບໍ່ເລີ່ມ, ຟ້າ=ກຳລັງເຮັດ, ຂຽວ=ສຳເລັດ, ແດງ=ລ່າຊ້າ.
Example:ຕົວຢ່າງ: Act 1.1.1 Training: Start 2026-03-01, End 2026-04-15, Status = In Progress.ກິດຈະກຳ 1.1.1 ຝຶກອົບຮົມ: ເລີ່ມ 2026-03-01, ສິ້ນສຸດ 2026-04-15, ສະຖານະ = ກຳລັງເຮັດ.

5. Budget5. ງົບປະມານ (Budget)

Purpose:ຈຸດປະສົງ: Cost each activity. Total = Unit Cost × Qty, summed into the Grand Total automatically.ຄິດໄລ່ຄ່າໃຊ້ຈ່າຍແຕ່ລະກິດຈະກຳ. ລວມ = ລາຄາ/ໜ່ວຍ × ຈຳນວນ, ບວກເຂົ້າຍອດລວມໃຫຍ່ໂດຍອັດຕະໂນມັດ.

Tip:ຄຳແນະນຳ: Activities & Resources come from the Logframe — edit only Unit, Unit Cost & Qty here.ກິດຈະກຳ ແລະ ຊັບພະຍາກອນ ດຶງມາຈາກ Logframe — ໃນນີ້ແກ້ສະເພາະ ຫົວໜ່ວຍ, ລາຄາ ແລະ ຈຳນວນ.

Example:ຕົວຢ່າງ: Training: Unit Cost 500 × Qty 4 = 2,000 USD.ຝຶກອົບຮົມ: ລາຄາ 500 × ຈຳນວນ 4 = 2,000 USD.

6. Risk (Risk Register)6. ຈັດການຄວາມສ່ຽງ (Risk Register)

Purpose:ຈຸດປະສົງ: An ISO 31000 risk register. Score = Likelihood × Impact (1–5), with a residual score after mitigation.ບັນຊີຄວາມສ່ຽງຕາມ ISO 31000. ຄະແນນ = ໂອກາດ × ຜົນກະທົບ (1–5), ພ້ອມຄະແນນຄົງເຫຼືອຫຼັງມາດຕະການ.

How to use:ວິທີໃຊ້:

  1. Click Add Risk Row, choose a Category and describe the risk.ກົດ ເພີ່ມຄວາມສ່ຽງ, ເລືອກໝວດ ແລະ ອະທິບາຍຄວາມສ່ຽງ.
  2. Pick Likelihood & Impact → the Inherent Risk score colours itself.ເລືອກ ໂອກາດ ແລະ ຜົນກະທົບ → ຄະແນນ ຄວາມສ່ຽງເລີ່ມຕົ້ນ ປ່ຽນສີເອງ.
  3. Write the mitigation, then set Residual Likelihood/Impact to show its effect.ຂຽນມາດຕະການ, ແລ້ວຕັ້ງ ໂອກາດ/ຜົນກະທົບ ຄົງເຫຼືອ ເພື່ອສະແດງຜົນຂອງມັນ.
Example:ຕົວຢ່າງ: "Currency volatility" — Inherent 5×4=20 (Critical) → after hedging, Residual 2×3=6 (Medium)."ຄ່າເງິນຜັນຜວນ" — ເລີ່ມຕົ້ນ 5×4=20 (ວິກິດ) → ຫຼັງປ້ອງກັນ, ຄົງເຫຼືອ 2×3=6 (ປານກາງ).

7. Tracking7. ຕິດຕາມຕົວຊີ້ວັດ (Tracking)

Purpose:ຈຸດປະສົງ: Record baseline, targets and actuals per year; progress % is calculated automatically.ບັນທຶກຄ່າພື້ນຖານ, ເປົ້າໝາຍ ແລະ ຜົນຕົວຈິງແຕ່ລະປີ; ຄິດໄລ່ % ຄວາມຄືບໜ້າໃຫ້ອັດຕະໂນມັດ.

How to use:ວິທີໃຊ້:

  1. Set Unit, Frequency, Responsible, Baseline and Overall Target.ຕັ້ງ ຫົວໜ່ວຍ, ຄວາມຖີ່, ຜູ້ຮັບຜິດຊອບ, ຄ່າພື້ນຖານ ແລະ ເປົ້າໝາຍລວມ.
  2. Add Year to log yearly targets/actuals; yearly actuals sum to Overall Actual.ກົດ ເພີ່ມປີ ເພື່ອບັນທຶກເປົ້າ/ຜົນລາຍປີ; ຜົນລາຍປີຈະບວກເປັນຜົນລວມ.
  3. Use ແຍກຍ່ອຍ (Disaggregation) for sex/age/location breakdowns.ໃຊ້ ແຍກຍ່ອຍ ສຳລັບແຍກຕາມ ເພດ/ອາຍຸ/ສະຖານທີ່.
  4. Set Status (Active/Completed/Cancelled) to archive old indicators.ຕັ້ງສະຖານະ (ພວມເຮັດ/ສຳເລັດ/ຍົກເລີກ) ເພື່ອເກັບຕົວຊີ້ວັດເກົ່າ.
Example:ຕົວຢ່າງ: Baseline 120, Target 500, Actual 300 → Progress 60%.ຄ່າພື້ນຖານ 120, ເປົ້າ 500, ຕົວຈິງ 300 → ຄວາມຄືບໜ້າ 60%.

8. PIRS8. PIRS (ໃບອ້າງອີງຕົວຊີ້ວັດ)

Purpose:ຈຸດປະສົງ: The Performance Indicator Reference Sheet — the official "definition" of each indicator.ຕາຕະລາງອ້າງອີງຕົວຊີ້ວັດ — "ນິຍາມ" ທາງການຂອງແຕ່ລະຕົວຊີ້ວັດ.

Locked columnsຖັນທີ່ລັອກ : Type/Level comes from the Logframe; Unit, Disaggregation, Baseline, Target, Frequency & Responsibility come from Tracking — edit those there.ປະເພດ/ລະດັບ ດຶງຈາກ Logframe; ຫົວໜ່ວຍ, ການແຍກຍ່ອຍ, ຄ່າພື້ນຖານ, ເປົ້າໝາຍ, ຄວາມຖີ່ ແລະ ຜູ້ຮັບຜິດຊອບ ດຶງຈາກ Tracking — ໃຫ້ແກ້ທີ່ນັ້ນ.

Edit hereພິມໄດ້ທີ່ນີ້ : Definition, Rationale, Data Source, Data Collection Method, How Calculation, Data Limitations/DQA.ນິຍາມ, ເຫດຜົນ, ແຫຼ່ງຂໍ້ມູນ, ວິທີເກັບຂໍ້ມູນ, ວິທີຄຳນວນ, ຂໍ້ຈຳກັດຂໍ້ມູນ/DQA.

Tip:ຄຳແນະນຳ: Click "Sync from Logframe" if new indicators don't appear.ກົດ "ດຶງຂໍ້ມູນຈາກ Logframe" ຖ້າຕົວຊີ້ວັດໃໝ່ບໍ່ປະກົດ.

9. Dashboard9. ແຜງຄວບຄຸມ (Dashboard)

Purpose:ຈຸດປະສົງ: A visual summary: budget distribution, indicator progress, and key totals. It refreshes from Budget & Tracking when you open it.ສະຫຼຸບແບບກາຟິກ: ການແບ່ງງົບປະມານ, ຄວາມຄືບໜ້າຕົວຊີ້ວັດ ແລະ ຍອດສຳຄັນ. ມັນອັບເດດຈາກ ງົບປະມານ ແລະ ຕິດຕາມ ເມື່ອເປີດ.

10. Saving, Exporting & Language10. ການບັນທຶກ, ສົ່ງອອກ ແລະ ພາສາ

  • Save (Ctrl+S) overwrites the open file; Download saves a new .json; Open loads one back.ບັນທຶກ (Ctrl+S) ຂຽນທັບໄຟລ໌ເດີມ; Download ບັນທຶກ .json ໃໝ່; Open ໂຫຼດກັບຄືນ.
  • Cloud keeps a quick backup online.Cloud ເກັບສຳຮອງດ່ວນອອນລາຍ.
